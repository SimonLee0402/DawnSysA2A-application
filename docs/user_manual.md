# Agent-to-Agent (A2A) 工作流平台 - 用户手册

## 开始使用

### 登录系统

1. 访问系统登录页面
2. 输入您的用户名和密码
3. 点击"登录"按钮进入系统

### 系统界面概览

登录后，您将看到本平台的界面，它主要围绕连接和编排智能体展开。以下是主要区域：

- **导航栏**：顶部导航菜单，包含主要功能入口
- **工作区**：中央内容区域，显示当前操作的详细内容
- **通知区**：显示系统消息和通知
- **用户菜单**：右上角用户头像，包含个人设置和退出选项

## 工作流管理

工作流是连接和编排智能体能力的强大工具。通过创建工作流，您可以定义一系列步骤，让不同的智能体按照预设的逻辑协同工作，自动化完成复杂任务。

### 工作流列表

工作流列表页面显示您有权访问的所有工作流：

- **我的工作流**：您创建的工作流
- **公共工作流**：系统中设置为公开的工作流
- **已执行工作流**：您已执行过的工作流实例记录

### 创建新工作流

1. 在工作流列表页面，点击"创建工作流"按钮
2. 输入工作流名称、描述和类型
3. 选择是否公开此工作流
4. 点击"创建"按钮进入工作流编辑器

### 管理现有工作流

在工作流列表中，每个工作流条目都提供以下操作：

- **查看**：查看工作流详情
- **编辑**：打开工作流编辑器修改工作流
- **执行**：创建并启动工作流实例
- **复制**：创建当前工作流的副本
- **删除**：删除工作流（不可恢复）

## 工作流编辑器

工作流编辑器是创建和修改工作流的核心功能区域，包含以下主要组件：

### 工作流编辑器界面

- **工具栏**：添加新步骤、保存工作流等操作
- **工作区**：编辑和可视化工作流步骤
- **属性面板**：配置当前选中步骤的参数
- **JSON编辑器**：高级用户可直接编辑工作流JSON定义

### 步骤类型

工作流编辑器支持以下类型的步骤：

#### A2A智能体步骤

此步骤允许您在工作流中调用已添加到平台的 AI 智能体来执行任务。您需要选择要调用的特定智能体，并配置发送给它的输入信息。

#### 条件步骤

根据条件评估结果，控制工作流执行路径：

- **条件表达式**：定义评估条件（支持变量、比较运算符、逻辑运算符）
- **满足条件时**：指定条件为真时跳转的步骤
- **不满足条件时**：指定条件为假时跳转的步骤

#### 循环步骤

重复执行特定步骤集合，支持两种循环类型：

- **ForEach循环**：遍历集合中的每个元素
  - 集合源：要遍历的数组或集合
  - 元素变量：当前元素的存储变量名
  - 索引变量：当前索引的存储变量名
  
- **While循环**：根据条件重复执行
  - 条件表达式：循环继续的条件
  - 迭代变量：记录当前迭代次数
  - 最大迭代次数：防止无限循环的安全措施

#### 数据转换步骤

处理和转换工作流中的数据：

- **Map转换**：将一个集合转换为另一个集合
- **Filter转换**：根据条件筛选集合中的元素
- **自定义转换**：使用表达式执行复杂的数据处理

### 变量和表达式

工作流中的变量使用`${变量名}`格式引用。系统支持以下变量类型：

- **字符串**：文本值
- **数字**：整数或浮点数
- **布尔值**：true/false
- **数组**：元素集合
- **对象**：键值对集合

表达式支持：

- **数学运算**：+, -, *, /
- **比较运算**：==, !=, >, <, >=, <=
- **逻辑运算**：&&, ||, !
- **路径访问**：`${data.user.name}`访问嵌套属性

### 保存工作流

编辑完成后，点击"保存工作流"按钮保存您的更改。系统会验证工作流定义，确保其有效。

## 工作流执行

### 创建工作流实例

1. 在工作流详情页或列表页点击"执行"按钮
2. 输入实例名称（可选）
3. 配置初始输入参数（如果有）
4. 点击"启动"开始执行

### 监控工作流执行

工作流实例页面显示实时执行状态：

- **基本信息**：实例名称、开始时间、当前状态
- **步骤列表**：所有步骤及其状态（待执行、执行中、已完成、失败）
- **上下文数据**：当前工作流上下文中的变量值
- **日志**：详细的执行日志

### 工作流实例操作

- **暂停**：暂停正在执行的工作流实例
- **恢复**：恢复已暂停的工作流实例
- **取消**：取消执行中的工作流实例
- **重试**：重试失败的工作流实例
- **查看结果**：查看已完成工作流的输出结果

## A2A智能体管理

本平台的核心是各种能够执行特定任务的 AI 智能体。在本节，您将学习如何将这些智能体添加到平台、管理它们的连接和凭证。只有将智能体添加到平台后，才能在工作流中调用它们的能力。

### 智能体列表

智能体列表页面显示系统中已配置的所有 A2A 兼容或通过本平台集成的智能体：

- **我的智能体**：您创建的智能体
- **公共智能体**：系统中公开的智能体

### 添加新智能体

1. 点击"添加智能体"按钮
2. 输入智能体名称、描述
3. 选择智能体类型（GPT、Claude、Gemini等）
4. 配置API密钥和端点
5. 点击"保存"按钮

### 测试智能体连接

添加或编辑智能体后，可以通过"测试连接"功能验证配置是否正确：

1. 在智能体详情页点击"测试连接"按钮
2. 系统将发送简单的测试消息
3. 显示响应结果和连接状态

### 管理智能体凭证

智能体凭证包含敏感信息，系统提供安全的凭证管理：

- **查看**：查看已保存的凭证信息（API密钥仅显示部分）
- **编辑**：更新API密钥和其他配置
- **删除**：移除智能体凭证

## 常见问题

### 工作流编辑器问题

**问题**：无法保存工作流，出现验证错误
**解决方案**：检查所有必填参数是否已配置，确保步骤引用的变量存在于上下文中

**问题**：条件步骤不按预期工作
**解决方案**：检查条件表达式格式是否正确，测试表达式是否能正确评估

### 工作流执行问题

**问题**：工作流执行失败，显示智能体调用错误
**解决方案**：检查智能体凭证是否正确，API密钥是否有效，网络连接是否正常

**问题**：循环步骤执行超时
**解决方案**：检查循环条件，确保有明确的终止条件，考虑增加最大迭代限制

### 账户问题

**问题**：无法登录系统
**解决方案**：检查用户名和密码是否正确，账户是否已激活，是否被管理员禁用

**问题**：忘记密码
**解决方案**：使用登录页面的"忘记密码"功能，按照邮件指引重置密码
